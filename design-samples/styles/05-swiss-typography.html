理解 Vue 3 组合式 API 的设计思想

2024.06.15 发布 · 阅读约 8 分钟 · 2,400 字 · #Vue #TypeScript

目录 CONTENTS

01 · 设计背景 02 · Options API 的局限 03 · 核心概念 04 · 实际应用 05 · 总结

Vue 3 的 Composition API 并非凭空出现，它是对 Vue 2 Options API 在使用中暴露出的痛点的一次系统性回应。

设计背景

在 Vue 2 时代，我们按照 data、methods、computed 等选项来组织组件逻辑。这种方式在小规模组件中清晰直观，但随着组件规模的增长，问题逐渐显现。

组合式 API 不是要取代 Options API，而是提供了一种更灵活的代码组织方式。两者可以共存。

// Vue 3 Composition API
const query = ref('')
const results = ref([])

watch(query, async (val) => {
  results.value = await fetchResults(val)
})

核心概念

Composition API 的核心思想是按逻辑关注点组织代码，而非按选项类型。

  • useSearch — 搜索逻辑封装
  • usePagination — 分页逻辑封装
  • useTheme — 主题切换封装

总结

Composition API 带给我们的不仅是语法上的变化，更是思维方式上的转变。它鼓励我们将逻辑视为一等公民。
